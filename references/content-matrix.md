# 通用内容变更影响矩阵

适用于非编码工作的变更——内容创作、文档更新、知识沉淀等场景。

本矩阵按**变更性质**分类，而非按项目分类，因此适用于所有类型的项目（银行CRM、视频制作、SEO营销、投资理财、社媒内容创作等）。

## 变更类型与影响文件

### 源材料变更

当输入材料（需求文档、参考资料、数据源等）发生变更时。

| 影响范围 | 需要检查的文件 |
|----------|----------------|
| 下游所有产出物 | 所有引用该材料的文档、报告、文章 |
| 引用该材料的方法论 | context/ 下的知识文件 |
| 基于该材料的决策记录 | memory/ 中的决策条目 |

**典型场景**：
- PRD 版本更新 → 检查所有基于旧版 PRD 的产出物
- 研究资料更新 → 检查基于旧资料的文章/报告
- 数据源变更 → 检查使用旧数据源的分析报告

### 模板/格式变更

当产出物模板、格式规范、样式指南发生变更时。

| 影响范围 | 需要检查的文件 |
|----------|----------------|
| 使用该模板的所有产出物 | outputs/ 下的相关文件 |
| 模板索引 | 记录可用模板的索引文件 |
| 格式规范文档 | context/format-specs.md 或等价文件 |

**典型场景**：
- 华夏银行公文格式变更 → 检查所有已生成的需求文档
- PPT 模板更新 → 检查历史 PPT 是否需要重制
- 文章风格指南更新 → 检查已发布/草稿文章

### 方法论/流程变更

当工作方法、分析框架、创作流程发生变更时。

| 影响范围 | 需要检查的文件 |
|----------|----------------|
| 使用该方法论的所有工作流 | 相关 skill 定义、commands/ |
| 方法论文档 | docs/methodology/、context/ |
| 操作手册 | CLAUDE.md 中的流程描述 |

**典型场景**：
- 视频制作流程优化 → 更新 video-studio 的工作流定义
- SEO 分析方法论更新 → 更新 seomachine 的分析方法
- 投资分析框架调整 → 更新 wealth-management 的方法论

### 领域知识更新

当行业知识、专业术语、法规政策等发生更新时。

| 影响范围 | 需要检查的文件 |
|----------|----------------|
| 引用该知识的所有内容 | context/、docs/ 下的相关文件 |
| 已发布/已交付的内容 | outputs/ 中引用旧知识的内容 |
| 术语表/知识库 | context/ 下的术语文件 |

**典型场景**：
- 银行监管政策变化 → 检查相关需求文档和分析
- SEO 算法更新 → 检查 SEO 策略和已发布文章
- 行业术语标准变更 → 更新术语表和相关内容

### 平台/渠道变更

当发布平台、分发渠道、目标媒介发生变更时。

| 影响范围 | 需要检查的文件 |
|----------|----------------|
| 该渠道的所有内容 | 发布配置文件、内容适配规则 |
| 排期/发布计划 | 相关的计划文件 |
| 平台配置 | config/platforms.yaml 或等价文件 |

**典型场景**：
- 新增发布平台（如从小红书扩展到抖音）
- 平台 API 变更 → 更新发布脚本
- 平台内容规范变更 → 检查内容适配规则

### 决策/方向变更

当项目方向、优先级、策略发生变更时。

| 影响范围 | 需要检查的文件 |
|----------|----------------|
| 受影响的所有相关项目 | 项目记忆、CLAUDE.md |
| 已进行的但被推翻的工作 | memory/ 中的相关记录 |
| 下游依赖此决策的内容 | 关联项目的知识文件 |

**典型场景**：
- 项目优先级调整 → 更新多个项目的记忆
- 技术选型变更 → 记录变更原因，更新相关文档
- 需求范围变更 → 检查已有产出物的有效性

### 工具/技能变更

当使用的工具、skill、Agent 发生变更时。

| 影响范围 | 需要检查的文件 |
|----------|----------------|
| 使用该工具的所有工作流 | skill 索引、工具配置 |
| 工具使用经验 | memory/ 中的工具经验记录 |
| 相关 commands/ | 命令定义文件 |

**典型场景**：
- 新安装 skill → 更新 skills.md、skills-sources.md
- skill 版本更新 → 检查使用经验是否仍然适用
- 工具被替代 → 更新所有引用该工具的文件

### 受众/品牌变更

当目标受众、品牌调性、用户画像发生变更时。

| 影响范围 | 需要检查的文件 |
|----------|----------------|
| 面向该受众的所有内容 | 用户画像文件、风格指南 |
| 已发布内容 | 检查是否需要适配新受众 |
| 品牌相关配置 | 语调/风格配置文件 |

**典型场景**：
- 受众从 B 端扩展到 C 端 → 调整内容和风格
- 品牌调性调整 → 检查所有面向外部的产出物
- 用户画像更新 → 影响内容创作策略

## 使用方式

在 Curate 的第二步（识别会话变更）中，当判断变更为内容相关时，读取本文件并按矩阵检查对应的文件是否需要更新。
