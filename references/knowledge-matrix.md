# 知识矩阵

定义所有知识类型、存储位置、治理策略和晋升路径。在 Curate 的第二步（识别会话变更）中用于分类知识，在第三步（执行治理）中用于选择治理策略。

## 知识类型

### 1. 领域知识

| 属性 | 值 |
|------|-----|
| **定义** | 特定领域的专业知识、术语、规范、方法论 |
| **示例** | 银行术语、CRM 概念、SEO 规则、投资方法论、视频制作规范 |
| **存储位置** | `context/`、`docs/methodology/`、`context/domain_knowledge/` |
| **变更矩阵** | content-matrix（领域知识更新 → 检查引用该知识的所有内容） |
| **评分权重** | 时效性 × 1.2（领域知识过时代价高） |
| **晋升路径** | context/ → CLAUDE.md（常用领域知识可晋升到项目约定） |
| **治理重点** | 准确性验证 + 去重 + 时效性 |

### 2. 项目决策

| 属性 | 值 |
|------|-----|
| **定义** | 项目方向选择、技术选型、优先级排序、方案取舍 |
| **示例** | "选择本地 Whisper 而非云服务"、"PRD v3.8 为当前基线" |
| **存储位置** | `memory/`、`CLAUDE.md` |
| **变更矩阵** | content-matrix（决策变更 → 检查受影响的所有相关项目） |
| **评分权重** | 可操作性 × 1.3（决策必须有明确行动指引） |
| **晋升路径** | memory/ → CLAUDE.md → 全局 CLAUDE.md |
| **治理重点** | 时效性检查 + 晋升 + 记录决策原因 |

### 3. 工具经验

| 属性 | 值 |
|------|-----|
| **定义** | 工具和 skill 的使用技巧、踩坑记录、最佳实践 |
| **示例** | "用 uv 不用 pip"、"markitdown 对 docx 无效用 python-docx"、"soffice 不可用" |
| **存储位置** | `memory/` |
| **变更矩阵** | change-matrix（工具变更影响编码）和 content-matrix（工具变更影响内容创作） |
| **评分权重** | 精简性 × 1.0（工具经验容易冗余，需要精简） |
| **晋升路径** | memory/ → CLAUDE.md |
| **治理重点** | Pattern-Key 去重 + 时效性（过时的工具经验要归档） |

### 4. 产出索引

| 属性 | 值 |
|------|-----|
| **定义** | 项目产出物的清单、版本记录、交付状态 |
| **示例** | 周报列表、架构图清单、需求文档版本、视频产出目录 |
| **存储位置** | `outputs.md` 或项目级索引文件 |
| **变更矩阵** | content-matrix（产出物变更 → 更新索引） |
| **评分权重** | 完整性 × 1.0（索引必须完整） |
| **晋升路径** | 输出索引 → 归档（历史产出物定期归档） |
| **治理重点** | 完整性检查 + 版本追踪 + 定期归档 |

### 5. 流程规范

| 属性 | 值 |
|------|-----|
| **定义** | 工作流步骤、排期计划、质量标准、发布流程 |
| **示例** | "每日 20:00 早报流程"、"SEO 内容生产全流程"、"视频制作管线" |
| **存储位置** | `CLAUDE.md`、`commands/`、`docs/` |
| **变更矩阵** | content-matrix（流程变更 → 检查使用该流程的所有工作） |
| **评分权重** | 一致性 × 1.2（流程必须自洽） |
| **晋升路径** | 命令 → CLAUDE.md（频繁使用的流程可晋升为项目约定） |
| **治理重点** | 一致性验证 + 与实际操作对照 |

### 6. 个人洞察

| 属性 | 值 |
|------|-----|
| **定义** | 行业趋势判断、方法论思考、专业视角、经验总结 |
| **示例** | "AI Agent 系统将向多 Agent 协作演进"、"银行数字化重点在数据治理" |
| **存储位置** | `memory/` |
| **变更矩阵** | content-matrix（洞察变更可能影响内容方向） |
| **评分权重** | 可操作性 × 1.1（洞察应可转化为行动） |
| **晋升路径** | memory/ → 全局 CLAUDE.md（跨项目适用的洞察） |
| **治理重点** | 可操作性检查 + 去重 + 验证洞察是否仍然成立 |

### 7. 编码知识

| 属性 | 值 |
|------|-----|
| **定义** | API 模式、代码规范、依赖管理、技术架构 |
| **示例** | "Python 3.10+ 语法不兼容 3.9"、"使用 Node.js 内置模块" |
| **存储位置** | `CLAUDE.md`、`docs/` |
| **变更矩阵** | change-matrix（编码变更的完整影响链） |
| **评分权重** | 完整性 × 1.2（编码知识缺失会导致重复踩坑） |
| **晋升路径** | memory/ → CLAUDE.md |
| **治理重点** | 完整性 + 变更影响追踪 |

## 跨类型交互

不同知识类型之间可能存在关联：

| 源类型 | 目标类型 | 关联场景 |
|--------|----------|----------|
| 领域知识 | 项目决策 | 基于领域知识的方案选择 |
| 工具经验 | 编码知识 | 工具使用涉及代码配置 |
| 项目决策 | 流程规范 | 决策导致流程变更 |
| 个人洞察 | 领域知识 | 洞察沉淀为领域知识 |
| 产出索引 | 项目决策 | 产出情况影响后续决策 |

在治理时，如果变更涉及某种知识类型，应检查是否同时影响了关联类型。
